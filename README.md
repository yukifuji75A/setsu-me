
# SETSU：Me
---


## 1. サービス概要

本サービスでは、性格や価値観、モチベーションの源泉など自分の特徴を入力するだけで、自分自身の **取扱説明書(トリセツ)** を作成することができます。
作成した **トリセツ** はリンク一つで共有可能。友人、恋人、新環境での自己紹介や、チーム内のスムーズな相互理解など、オンオフを問わないあらゆるコミュニケーションをサポートします。

---

## 2. このアイデアが生まれた背景

近年、若者世代を中心にMBTIをはじめとした性格診断が広く流行しており、SNS上でも **「自己理解」** や **「自己の言語化」** への関心が急速に高まっています。

こうした中で、「単に診断して終わるのではなく、自分の特徴を客観的に可視化し、周囲に共有できるツールがあれば便利なのではないか」と考えました。

---

## 3. 課題の整理（表面的になっていないか）

### 3-1. 表に見えている困りごと:

**新しい環境で相手に本当の自分を伝えることができない、知ってもらうことができない** 

### 3-2. 本当に解決したい課題は何か

* **自己紹介のネタ不足:** 新しい環境での自己紹介がいつも定型文になってしまい、自分の本当の持ち味やキャラクターを上手く伝えられない。

* **相互理解の時間ロス:** 新しい職場やコミュニティで、お互いの仕事の進め方の癖、得意・不得意、不調のサインを把握するまでに数ヶ月以上の時間がかかってしまう。

* **地雷や境界線の伝えづらさ:** 「これをされるとモチベーションが下がる」「こういう時にストレスを感じる」という繊細な部分を、相手に重く捉えられずに伝える手段がない。

* **言語化の壁:** 自分の活かし方やストレス要因を、自分一人では客観的に言語化できない。

#### 本当に解決したい課題：**自己の言語化が苦手な人でも、新しい環境で本当の自分を伝えれるようにしたい！**

---

## 4. 想定ユーザー
**新しい環境に飛び込む人・自己開示に不安がある人**
* 転職、異動、入学、新しいコミュニティへの参加直後で、「自分をどう紹介すればいいか分からない」「馴染むまでに時間がかかる」と悩んでいる人。
* 自分の強みや不調のサイン、言葉にしにくい「地雷（されて嫌なこと）」を、空気を壊さずにスマートに伝えたい人。

---

## 5. 競合調査

### 5-1. 似たサービスの調査

- サービス名：キャリタスQUEST
- URL：https://quest.career-tasu.jp/
- 特徴: 就活支援サービスで、20問の質問に答えると「100種類以上のキャラクター」から自分の性格タイプが割り出され、RPGのステータス画面のようなビジュアルで強み・弱みが可視化される。

- サービス名：16Personalities
- URL：https://www.16personalities.com/ja
- 特徴：自分の性格タイプが「建築家」「冒険家」などのキャラクター（アバター）と、パーセンテージのグラフで可視化される。

- サービス名：BaseMe
- URL：https://baseme.app/
- 特徴：日々の活動や経験、興味関心のプロフィール情報を読み込ませることで、AIが「自分だけのパーソナルAI」を構築し、自己分析やキャリア傾向をサポートしてくれるツール。

---

### 5-2. それでも自分が作りたい理由

既存の性格診断やプロフィールの共有ツールの多くは、「診断結果を眺めて終わり」であったり、ビジネスに寄り添いすぎていてプライベートでは使いにくかったりします。
私自身、新しい環境や人間関係の中で「自分のトリセツをあらかじめ渡せたら、もっとお互い生きやすくなるのに」というもどかしさを強く感じていました。

だからこそ、既存の枠組みにとらわれず、個人の価値観や日々の変化までをグラフィカルに「組み立てる」ことができ、オンオフ問わずリンク一つでカジュアルに共有できる、**本当に自分が使いたいと思えるプロダクトを形にしたい**と考え、開発を決意しました。

---

## 6. このサービスで提供したい価値

* **言葉にできない「自分」の言語化・構造化**
 -感覚的になりがちな自分の特性や好不調のサインを、いくつかの質問に答えるだけで綺麗な「仕様書」として論理的に整理できます。

* **自己開示の心理的ハードルをゼロに**
 -「私の地雷はこれです」「こう扱ってほしい」という直接言うと角が立つことも、トリセツという遊び心あるフォーマットを介すことで、スマートかつ心理的抵抗なく相手に伝えられます。

* **相互理解にかかる時間の大幅なショートカット**
 -初対面やチーム結成初期の「探り合う時間」をなくし、リンク一つを共有するだけで、お互いの活かし方や最適な距離感が初めからクリアになります。

* **日々の変化に寄り添う「生きているプロフィール」**
 -一度作ったら終わりの静的な診断ではなく、経験や環境の変化、今の気分に合わせていつでもアップデートし、常に「最新の自分」をストックしておけます。

## 7. このアプリで実現すること

### MVPで実装する機能

#### 1. 認証・ユーザー管理機能
* **ログイン**
* **ユーザー登録:** アカウント作成 -> プロフィール設定 -> 共通の基本情報入力

#### 2. トリセツ作成機能
* **トリセツ作成画面（デフォルト）:** 共通の基本情報に加え、詳細な追加情報を入力。
* **AIトリセツ文章生成:** 入力データを元に、AIが「取扱説明書風の解説文」を自動生成して保存。

#### 3. マイページ
* **画面内タブ切り替え:** ページ遷移をせず、その場のタブクリックで「基本情報」「AI分析文章」を切り替え可能。

#### 4. トリセツ管理画面
* **プレビュー表示:** 作成して保存したトリセツのプレビューを表示。

---

### 本リリースで実装する機能

#### 1. LINE連携
* **LINEアカウント認証:** LINEアカウントを使った簡易新規登録およびログイン機能。

#### 2. 外部共有機能
* **共有用URLの発行:** テーマ別の管理画面から、相手に送るための閲覧専用URLをコピーする機能。
* **トリセツ共有ページ閲覧機能:** 共有URLを開くだけで、ログイン不要で第三者が「あなたの取扱説明書」を閲覧できる機能。
* **LINE共有機能:** [LINEで共有]ボタンをクリックすると閲覧中のページURLをLINE上でシェアできる機能。

#### 3. マルチテーマ機能
* **テーマ選択（恋人 / 友だち / デフォルト）:** 相手に合わせた専用のトリセツを個別に作成・管理可能。
* **シチュエーションに応じた構造的差別化:** 単なる色・フォントの切り替えではなく、見せる相手に合わせてレイアウトや全体の構造を根本から切り替える。
  * **恋人:** 親密な関係性に適した「パーソナルブック」風構造
  * **友だち:** カジュアルに楽しく自己開示ができる「プロフカード」風構造
  * **デフォルト:** 誰にでも見せられる客観的な「製品マニュアル」風構造

#### 4. 編集機能
* **トリセツの再編集（URL維持）:** 作成済みのトリセツデータを後からいつでも編集・上書き可能（編集・更新しても共有URLは変わりません）。
* **AI生成文章のユーザー編集・再生成:** AIが生成した文章を自分の言葉で自由に修正できる機能、およびワンタップで別の切り口で作り直す「再生成ボタン」の設置。

#### 5. ビジュアルの強化
* **レーダーチャート（グラフ）表示:** 特性やパラメーターを可視化するグラフ機能を実装。マイページや共有ページのタブ切り替え項目に追加され、選んだテーマ（世界観や配色）にデザインが完全同期。
* **プロフィール画像の自由アップロード:** 初期設定の簡易アバターだけでなく、自分の好きな写真やイラストをアイコンとして自由にアップロードできる。

## 8. 懸念点とその対策

### 懸念点①：入力項目の多さによる離脱（入力が面倒）
* 問題点:深いトリセツを作るには入力項目が多くなり、ユーザーが途中で飽きてしまう。

* 対策:
  * 初回は「3つの必須質問」だけで3分で初期トリセツ（Ver. 1.0）が完成する導線を設計。
  * 「後からパーツを追加してトリセツをアップデートしていく（Ver. 1.1...）」という、段階的な入力体験（ゲーミフィケーション）を取り入れる。

### 懸念点②：データ不足によるAI分析の「薄さ」
* 問題点:入力項目が少ない段階だと、AIの出力する取扱説明が抽象的でつまらない内容になってしまう。

* 対策:
  * プロンプト（AIへの指示）を「取扱説明書」のフォーマットに厳格に特化させ、少ない情報からでも「ユニークなトリセツの文体」として楽しめる工夫をする。
  * 現在の入力度を「マニュアル完成度：45%（追加情報でAI分析がよりシャープになります）」のように表示し、追記を促す。

### 懸念点③：API消費によるLLMのコスト増加
* 問題点:ユーザーが1文字書き換えるたびにAI分析が走ると、APIコストが爆発する。

* 対策:
  * 「トリセツを発行する（更新する）」という確定ボタンを押したタイミングのみAI分析を実行する方式を採用。
  * 前回の分析から一定以上の情報の変更（例：変更率30%以上）がない場合は、再生成ボタンを制限する。

## 9. 今後の展開・発展の方向性

1. **グラフィカルな「仕様書」デザイン:**
* 自動車や家電の製品マニュアル、あるいはSFチックな「スペック表」のような、遊び心のあるUIテーマの切り替え機能。

2. **ソーシャル・コミュニケーション機能:** 
* 2人のトリセツを掛け合わせる「相性・バグチェック（コミュニケーションの衝突ポイント予測）」機能。
* チームメンバーのトリセツを一覧化し、検索できる「チームマニュアル」機能。

3. **トリセツの「同封」機能:**
* 就職活動（選考）や、マッチングアプリ、SNSのプロフィールにリンクとして貼り付け、スマートに自己開示できる外部連携。

## 10. 技術スタック

### 10-1. 使用予定の技術

| カテゴリ | 技術 | 用途 |
|----------|------|------|
| バックエンド | Ruby on Rails 7.2.X | サーバーサイド開発、API連携、CRUD処理 |
| フロントエンド | Tailwind CSS + DaisyUI / JavaScript | UI構築、テーマに応じたコンポーネント・レイアウト制御 |
| DB | PostgreSQL | ユーザー情報・共通基本情報・テーマ別データの管理 |
| 認証 | Devise | ユーザー登録・ログイン認証 |
| AI | OpenAI API or Gemini API | テーマ別のトリセツ文章生成・マイページに表示する分析レポート生成 |
| グラフ可視化 | Chart.js  | ユーザー情報の可視化 |
| 画像管理 | Active Storage | プロフィール画像管理 |
| インフラ | Docker | 開発環境 |
| デプロイ先 | Render | 本番環境 |

---

### 10-2. 技術選定の理由

#### バックエンド：Ruby on Rails 7.2.X
* 認証・DB・API連携などWebアプリ開発に必要な機能が揃っている
* 開発速度が速く、全テーマ共通の基本情報とテーマ別の追加入力データを組み合わせる複雑なロジックを柔軟に実装できる
* 卒業制作での採用実績が多く、情報が豊富

#### フロントエンド：Tailwind CSS + DaisyUI
* 独自パーツの多い「トリセツ」の画面を、素早く構築する
* 本リリース時のマルチテーマ機能（パーソナルブック、プロフカード、製品マニュアル）において、各テーマに応じたコンポーネントの構造的切り替えと、Tailwindの柔軟なスタイリングを組み合わせることで、破綻のないUI実装が可能になる

#### データベース：PostgreSQL
* Railsとの相性が良く、Renderなどの主要PaaSで標準対応している
* 本番環境でも十分な性能を持ち、データの管理・運用が安全に行える

#### 認証：Devise
* Rails認証の定番であり、ログイン・会員登録機能を短期間で安全に実装可能

#### AI分析：OpenAI API / Gemini API
* ユーザー情報から「取扱説明書」風のユニークな解説文を自動生成するために必須
* コストやテーマ別の文体出し分け（恋人なら少し砕けた文体、デフォルトなら取扱説明書風の硬い文体など）の精度を検証して選定

#### グラフ可視化：Chart.js (直接使用)
* ユーザーの情報を可視化するレーダーチャートの実装に使用。

#### 画像管理：Active Storage (Rails標準機能)
* プロフィール（アイコン）画像の保存
* 外部ライブラリの追加なしで、Railsの標準機能としてシンプルに実装可能

#### デプロイ先：Render
* Railsとの相性が良く、GitHub連携による自動デプロイが簡単
* プログラミングスクールや個人開発での実績・ナレッジが多くトラブル対応がしやすい
